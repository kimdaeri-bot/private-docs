Vision & Mission

“Travel intelligence, distilled in our labs, distributed as a platform.”

VISION

“오퍼레이션으로 검증한 인텔리전스만, SaaS 플랫폼으로 유통.”

전 세계에 흩어진 여행 인벤토리와 도메인 노하우를 단일 인터페이스로 통합. 어떤 중소 여행사라도 종합 여행사 수준의 오퍼레이션 가능.

MISSION

“오퍼레이션으로 검증한 것만 유통.”

외국 플랫폼 컨설팅 + 자체 OTA(엔틱스 · 크루즈링크 · 런던틱스 · 브로드웨이쇼)에서 검증된 기능만 SaaS로 외부 공급. 책상에서 만든 SaaS가 아닌, 현장에서 단련된 SaaS.

PRINCIPLE

“카테고리 정의 — Travel Intelligence Platform.”

Salesforce가 CRM을, HubSpot이 Inbound Marketing을, Snowflake가 Data Cloud를 정의했듯, Travel Intelligence Platform 카테고리를 시장에 선점.

Phase 3 · Solution — Travel Intelligence Platform. 개발 중 · 2027 베타

여행사용 SaaS 플랫폼 + API Aggregator + 3 AI Intelligence Layers.

A · SAAS 플랫폼

여행사 전용 SaaS 플랫폼.

중소 여행사가 자체 상품과 우리 API 인벤토리를 하나의 사이트에서 통합 판매. 화이트라벨 · 자체 도메인 · 결제 · 정산 · CS 통합.

B · API AGGREGATOR

통합 인벤토리, 단일 API로.

크루즈 · 료칸(Rakuten) · 유럽철도 · 앤실러리 · 공연 — 5+ 카테고리를 Single API로 정규화. 한 번 연결로 모든 카테고리 판매.

C · AI LAYERS

차별화는 AI에서.

Design AI · Parsing AI · Vibe Coding — 디자인 · 등록 · 편집 모두 자동화. 일반 SaaS는 템플릿 판매, 우리는 인텔리전스 판매.

현황 Cruiselink가 SaaS 모델의 1차 파일럿(LIVE). 전체 Travel Intelligence Platform은 현재 개발 중 — 2027 베타 출시, 2028 정식 GA 목표.

Moats — 4가지 해자.

후발 SaaS가 따라올 수 없는 4가지 이유.

01

10년 도메인 자산 해자

컨설팅(Phase 1) + 자체 운영(Phase 2)으로 쌓은 10년치 도메인 노하우. 책상 위 경쟁사와 AI 학습 데이터의 질이 근본적으로 다름.

02

카테고리 정의 해자

“Travel Intelligence Platform” 카테고리 글로벌 부재. 먼저 정의하면 표준화 — Salesforce / HubSpot / Snowflake 패턴.

03

양면 네트워크 해자

공급자(Rakuten · 크루즈 · 철도 · 앤실러리 · GDS) ↔ 여행사 양쪽 동시 락인. 한쪽이 늘면 반대쪽도 증가하는 양면 마켓플레이스.

04

Labs 해자

“오퍼레이션으로 검증한 것만 유통” — 책상에서 만든 SaaS와 다른 현장 검증 SaaS. 경쟁사가 흉내내려면 컨설팅 + OTA를 모두 운영해야 함.
