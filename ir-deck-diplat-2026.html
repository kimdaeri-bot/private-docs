03 / 15 — Problem

여행 산업의 API 파편화

수백 개의 API가 난립 — 각각 다른 인증, 포맷, 정산

각 API마다 필요한 것

  • 별도 법적 계약 협상
  • 3~6개월 개발 기간
  • 다른 데이터 포맷 정규화
  • 각각의 정산 시스템 구축
  • 지속적인 유지보수

파편화된 API 예시

분야API 수
항공Amadeus, Sabre, 직접 50+
호텔Booking, Expedia, ETG 등
크루즈Widgety, 각 선사 직접
철도Railclick, SBB, JR, 12go
티켓Tixstock, Tiqets, LTD
료칸Rakuten RTX (6,600개)
