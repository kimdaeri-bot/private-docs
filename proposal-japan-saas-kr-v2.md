## 📄 Slide 5: 여행사의 Pain Point

**① 크루즈 웹사이트 구축의 어려움**
- 크루즈 특화 웹사이트 개발은 일반 여행사이트와 완전히 다름
- 선사별 상이한 API·예약 시스템을 통합하려면 **초기 개발비 3,000~5,000만엔** 소요
- 유지보수·서버·보안 등 **월 100~300만엔** 지속 비용
- 개발사에 크루즈 도메인 지식이 없어 기획부터 재작업 반복

**② 크루즈 상품 콘텐츠 제작 병목**
- 선사마다 일정·요금·선박 정보를 **수동으로 수집·번역·등록**해야 함
- 하나의 크루즈 상품 페이지 작성에 **2~4시간** 소요 (기항지 정보, 일정표, 요금표, 사진)
- 9개 선사 × 수백 개 상품 = **콘텐츠 제작에만 연간 수천만엔의 인건비**
- 프로모션·요금 변경 시 **실시간 업데이트 불가** → 기회 손실

**③ 선사별 개별 대응 부담**
- MSC·Princess·HAL·NCL 등 각 선사의 예약 시스템·약관·절차가 상이
- 담당자가 선사별로 **별도 교육·운영** 필요
- 견적 확인 → 작성 → 고객 전달까지 **평균 2~3일**

**④ 다국어 대응 한계**
- 일본어 고객 응대에도 인력 집중, 영문 선사 자료 해독에 추가 리소스
- 한국·중국·영문 고객 대응은 사실상 불가

**⑤ 24시간 대응 불가**
- 야간·휴일 문의 응답 불가 → 리드 손실
- 경쟁사 온라인 서비스에 고객 이탈

## 📄 Slide 6: CruiseLink SaaS 서비스 소개

**여행사의 브랜드로, 크루즈링크의 기술로**

CruiseLink SaaS는 크루즈 전문 여행사에 **맞춤형 화이트라벨 웹서비스**를 제공합니다.

- ✅ 여행사 **자체 도메인·브랜드**로 운영
- ✅ 9개 선사 **실시간 크루즈 데이터** 자동 연동
- ✅ 상품 콘텐츠 **자동 생성** — 수작업 제로
- ✅ AI 챗봇으로 **24시간 고객 응대**
- ✅ 일본어·영어·한국어 **다국어 지원**
- ✅ 초기 개발 없이 **4~8주 내 오픈**

## 📄 Slide 8: 화이트라벨 웹사이트

**여행사 브랜드 100% 노출**

- 여행사 **자체 도메인** 운영 (예: cruise.travel-xxx.co.jp)
- **여행사 로고·컬러·디자인** 커스텀 적용
- CruiseLink 로고는 표시하지 않거나, "Powered by CruiseLink"로 최소 표기
- **반응형 웹** — PC·태블릿·스마트폰 대응
- SSL 보안 인증서 포함
- **SEO 최적화** — 구글·야후·빙 검색 대응
- **페이지 구성:**
  - 메인 홈 (추천 크루즈·프로모션)
  - 크루즈 검색 (선사·목적지·출항일 필터)
  - 크루즈 상세 (일정·선박·요금·기항지)
  - 견적 요청 / 온라인 예약
  - 기항지 가이드·크루즈 팁
  - 회사 소개·문의

## 📄 Slide 10: 자동 콘텐츠 생성 시스템

**Pain Point 해결: 수동 콘텐츠 제작 → 자동화**

- 크루즈 상품 페이지 **자동 생성** — 일정표, 기항지 정보, 요금표, 선박 정보
- **SNS용 콘텐츠** 자동 생성 — 인스타그램·네이버 블로그 포맷
- 기항지 가이드·크루즈 팁 페이지 **자동 작성**
- 프로모션·요금 변경 **실시간 반영**
- 일본어 **자동 번역** 포함

| 기존 (수동) | 도입 후 (자동) |
|-------------|---------------|
| 상품 1개당 2~4시간 | 자동 생성 (0분) |
| 연간 수천만엔 인건비 | 시스템 자동 처리 |
| 실시간 업데이트 불가 | 일 단위 자동 갱신 |

- 일본어 자연어 질문 → 최적 크루즈 상품 추천

- "예산 50만엔 이하 알래스카 크루즈" → 조건 맞춤 추천
